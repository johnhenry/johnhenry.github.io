HTML Tags for Data

John Henry

As web developers, we often ignore the semantics of HTML tags. We arbitrarily interchange divs, spans, and sections as generic containers. We might follow up an h3 tag with a few h1s to achieve a desired style.

It’s great that we can do this on the web without having our browsers throw errors like <div> in <li> *not allowed*. aborting render. upsetting user.; but when displaying data in documents, it may be useful
